Task 1 & 2: Analysis Foundation and Bayesian Change Point Modeling
Task 1 & 2 are assessed based on observable evidence in the repository (merged because both tasks form the core analytical pipeline leading from data understanding to modeling): Task 1 Components: 1. Analysis Workflow Document: A document or notebook section outlining planned steps from data loading to insight generation is present 2. Event Dataset: A structured CSV or tabular file with at least 10–15 key events (geopolitical, OPEC, economic) and their approximate dates is committed 3. Assumptions and Limitations: Documentation acknowledging the correlation-vs.-causation constraint is present Task 2 Components: 4. EDA Notebook: A Jupyter notebook contains loading and visualization of the Brent oil price series (raw prices and log returns) with trend, stationarity, and volatility analysis 5. PyMC Model Implementation: Notebook includes a Bayesian change point model built using PyMC with a defined switch point (tau), before/after parameters, and a likelihood connected via pm.math.switch 6. MCMC Sampling and Convergence: Evidence of MCMC sampling (e.g., pm.sample() call) with convergence diagnostics such as r_hat values and trace plots 7. Change Point Interpretation: Notebook includes posterior distribution plots for tau and before/after parameters, with written interpretation linking detected change points to specific events from the compiled event dataset
